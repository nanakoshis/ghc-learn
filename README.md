上手く活用するためのテクニック

・関連ファイルを開いておく

GitHub Copiltは、エディターで現在開いているファイルを参考にサーバーに送るプロンプトを生成するため、より的確な提案を受けることが可能

コンテキスト変数#file: を用いることでファイルを添付することも可能

・適切なコメントを入れる

ファイルの最初にファイル内で行われる全体的な処理のコメントを記載

処理の流れ

インプットアウトプット

関数の前に関数で行われる処理を記載

関数内の変数や、呼び出す関数に利用目的や処理概要などのコメントを記載

・意味のある命名をする

関数名が処理を表す名前にすることで適切な処理の提案を受けることが可能

・ライブラリバージョンを指定

具体的なライブラリ名やバージョン指定することでより意に沿った回答を得られやすくなる

・コンテキストが変わる場合、新規チャット

会話の流れが変わる場合は、新規チャットを開く

便利なショートカット



エージェント機能

GitHub Copilot Chatにおいて質問範囲を限定する

@workspace : ワークスペース内のコードやファイルに関連する質問

@terminal : 統合ターミナルに関連する質問

@vscode : VSCode（エディタ）に関連する機能の質問

スラッシュコマンド

@workspace関連

/doc ドキュメントコメントを追加　ドック

/explain 指定した範囲のコードが実行される際の動作を説明　エクスペイン

/fix 選択したコードの問題点、修正点の提案　フィックス

/egnerate プロンプトに沿ったコードを生成　ジェネレート

/optimize 選択したコードのパフォーマンス改善　オプティマイズ

/tests コードの単体テストの作成

/new 入力した自然言語に基いた新規のワークスペースの作成



@vscodeエディタに対するコマンド

/search エディタの検索機能でワークスペース内のファイルを検索

/api エディタの拡張機能の開発に関する回答を生成

@terminalに対するコマンド

/explain ターミナルで実行された処理の内容を説明

GitHub Copilot Chatに関連するコマンド

/help Copilot Chatの使用に関するヘルプを表示する

/clear 現在のチャットを終わらせて新規チャットを開始する



コンテキスト変数

ソースコード

#codebase 現在のワークスペースの内容すべて
ワークスペース内のファイルとフォルダに関する情報、およびそのワークスペース固有の設定や構成

#editor 現在選択しているエディターで表示されているソースコード

#file:{FILE_NAME} ワークスペース内の指定したファイル

ターミナル

#terminalLastCommand アクティブなターミナルで最後に実行したコマンドの結果
エラー分析などに有効

#terminalSelect アクティブなターミナルで選択した文字列が追加

Git

#git 現在開いているGitリポジトリに関する情報を追加
